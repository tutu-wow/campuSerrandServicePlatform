# <font style="color:#48b378;">二.功能需求分析</font>
校园跑腿管理系统面向学生、跑腿者、管理员三个角色进行功能需求分析，解决传统跑腿管理模式信息传递碎片化、流程低效、沟通不畅等问题。

## 2.1学生需求分析
（1）登录：学生用注册账号密码登录系统，未登录仅可浏览首页、公告及部分公开信息，登录后可进行发布订单、查看订单、在线沟通、投诉及评价等操作。

（2）发布跑腿订单：学生可根据需求发布订单，填写取送地址、物品说明、联系方式、报酬及备注，提交后等待接单。

（3）查看订单信息：学生可查看自身订单详情、当前状态，以及跑腿者信息和服务路线，掌握订单进度。

（4）在线沟通：订单执行中，学生可通过聊天功能与跑腿者实时沟通，确认订单细节，提升服务效率。

（5）投诉与评价：订单结束后，学生可评价跑腿者服务；若有问题，可提交投诉维护自身权益。

（6）社区互动：学生可浏览、评论、收藏社区帖子，分享经验、交流校园信息，提升平台活跃度。

（7）个人中心：学生可以修改自己的个人信息，发布的帖子，查看评价、投诉、收藏以及信誉分记录。

## 2.2 跑腿人员需求分析
（1）登录与认证：跑腿人员用注册账号登录后，提交证件照片等实名认证资料，经管理员审核通过即可获得接单权限。

（2）浏览与接单：跑腿人员可在订单大厅浏览待接订单，结合订单类型、取送位置、报酬等条件挑选接单。

（3）订单处理：跑腿人员接单后，可更新订单状态，上传订单状态，方便学生掌握配送进度。

（4）在线沟通：跑腿人员可与学生在线核对取送、备注等信息，减少偏差。

（5）收入管理：跑腿人员可查看接单记录、收入明细及信誉分变动。

（6）提现申请：跑腿人员账户满足提现条件后，可提交申请，经管理员审核完成收益结算。

（7）评价反馈：订单完成后，跑腿人员可评价学生。

## 2.3 管理员需求分析
（1）用户管理：管理员可对平台内的学生用户、跑腿人员及管理员账号进行统一管理与维护。

（2）跑腿者认证管理：管理员负责审核跑腿人员提交的实名认证资料。

（3）订单审核与管理：管理员可审核学生发布的跑腿订单，核查订单内容是否符合平台规范；同时可查看订单详情、处理异常订单、取消违规订单及指派配送人员。

（4）投诉处理：管理员负责受理用户发起的投诉申请，查看投诉缘由与相关凭证，并依据实际情况做出退款、赔付或驳回投诉等处理结果。

（5）评价与信誉管理：管理员可查看订单相关评价内容，对不当评价进行处理，结合服务履约情况管理用户信誉分值。

（6）公告管理：管理员可对系统公告进行发布、编辑与删除操作。

（7）社区内容管理：管理员负责审核社区内帖子、评论等互动内容，清理违规信息。

（8）收入与提现管理：管理员可查看跑腿人员的收入明细，对提现申请进行审核与处理。

（9）系统配置管理：管理员可设置与维护平台运行参数，包括最低接单信誉分、最大接单数量、报酬区间、服务时段、订单超时规则及投诉处理时效等。

（10）自动审核配置管理：管理员可对AI自动审核功能进行参数配置，包括功能启用状态、审核提示词设定等。

（11）数据统计分析：管理员可通过数据统计分析功能查看平台运营状况，如用户规模、订单总量、营收情况、投诉数据等。
